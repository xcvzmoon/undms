# Release v1.4.0

This release note documents the current packaging and publishing setup for `undms`, along with the extraction, similarity, OCR, and benchmarking improvements included in v1.4.0.

## Highlights

### Performance improvements

- Reduced extraction overhead by removing shared `DashMap` aggregation in favor of direct parallel collection
- Precomputed reference metadata for similarity operations instead of rebuilding it for every document/reference comparison
- Removed double parsing in both DOCX and PDF extraction paths
- Reduced string and row allocation overhead in text, PDF, and XLSX processing
- Added a UTF-8 fast path for text extraction before charset detection
- Improved OCR throughput by processing image variants lazily and exiting early when a strong result is found

### Reliability improvements

- Fixed Unicode-aware similarity normalization by using character counts instead of byte lengths
- Replaced brittle DOCX unit tests with in-memory generated fixtures
- Preserved image metadata even when OCR/model initialization fails
- Extraction error logging is now opt-in through `UNDMS_LOG_ERRORS=1`

### Benchmark workflow

- Benchmarks are now split into focused commands for faster iteration:
  - `pnpm bench:text`
  - `pnpm bench:similarity`
  - `pnpm bench:documents`
  - `pnpm bench:image`

### Distribution model

- The root `undms` package stays small and does not bundle native `.node` files directly
- Native binaries are distributed as scoped platform-specific optional packages generated by `napi-rs`
- CI builds, tests, assembles, and publishes those platform packages before publishing the root package

### Release workflow

- Releases are published from GitHub Actions on pushes to `main`
- Publishing is triggered by release commits in the format `chore(release): bump x.x.x`
- Pre-release commit messages such as `chore(release): bump x.x.x-beta.0` publish with the `next` tag

### Local development

- `pnpm test` builds the native binding before running AVA
- CI uses a separate binding validation path so downloaded artifacts are tested directly

## Installation

```bash
bun add undms
# or
npm install undms
# or
pnpm add undms
```

## Usage

### CommonJS

```js
const { extract } = require('undms');
```

### ESM

```js
import { extract } from 'undms';
```

## Platform Support

Pre-built binaries for:

- Windows x64
- macOS x64 (Intel)
- macOS ARM64 (Apple Silicon)
- Linux x64 (glibc)
- Linux x64 (musl)
- Linux ARM64 (glibc)

## Requirements

- Node.js 20+ or Bun

## Release Process

1. Update the version as needed.
2. Create a release commit such as `chore(release): bump 1.3.0`.
3. Push the commit to `main`.
4. GitHub Actions publishes all generated platform packages, then publishes the root `undms` package.
