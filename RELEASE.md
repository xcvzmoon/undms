# Release v1.2.0

This release note also documents the current packaging and publishing setup for `undms`.

## Highlights

### Distribution model

- The root `undms` package now stays small and does not bundle native `.node` files directly
- Native binaries are distributed as scoped platform-specific optional packages generated by `napi-rs`
- CI builds, tests, assembles, and publishes those platform packages before publishing the root package

### Release workflow

- Releases are published from GitHub Actions on pushes to `main`
- Publishing is triggered by release commits in the format `chore(release): bump x.x.x`
- Pre-release commit messages such as `chore(release): bump x.x.x-beta.0` publish with the `next` tag

### Local development

- `pnpm test` now builds the native binding before running AVA
- CI uses a separate binding validation path so downloaded artifacts are tested directly

## Installation

```bash
bun add undms
# or
npm install undms
# or
pnpm add undms
```

## Usage

### CommonJS

```js
const { extract } = require('undms');
```

### ESM

```js
import { extract } from 'undms';
```

## Platform Support

Pre-built binaries for:

- Windows x64
- macOS x64 (Intel)
- macOS ARM64 (Apple Silicon)
- Linux x64 (glibc)
- Linux x64 (musl)
- Linux ARM64 (glibc)

## Requirements

- Node.js 20+ or Bun

## Release Process

1. Update the version as needed.
2. Create a release commit such as `chore(release): bump 1.2.1`.
3. Push the commit to `main`.
4. GitHub Actions publishes all generated platform packages, then publishes the root `undms` package.
